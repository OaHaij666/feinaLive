# 弹幕处理流程

## 概述

弹幕处理是整个系统的核心功能之一，负责接收、解析、分发B站直播间的弹幕消息，并触发相应的业务逻辑。

## 详细步骤说明

### 步骤 1: B站弹幕服务器连接

```
┌─────────────────────────────────────────────────────────────────┐
│                  B站弹幕服务器连接流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 应用启动时:                                                 │
│     │                                                           │
│     ▼                                                           │
│  BilibiliClient(room_id=直播间ID)                              │
│     │                                                           │
│     │ 创建 WebSocket 连接                                        │
│     ▼                                                           │
│  wss://broadcastlv.bilibili.com:443/sub                       │
│     │                                                           │
│     │ 发送认证包 (roomid, key)                                  │
│     ▼                                                           │
│  认证成功，开始接收弹幕                                          │
│                                                                 │
│  2. 接收到的弹幕类型:                                           │
│     - 弹幕消息 (DANMU_MSG)                                     │
│     - 礼物消息 (SEND_GIFT)                                     │
│     - 进入欢迎 (WELCOME)                                        │
│     - 关注消息 (INTERACT_WORD)                                  │
│     - 心跳包 (HEARTBEAT)                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 步骤 2: 消息解析

B站弹幕协议使用 JSON 格式，但字段名较长，客户端将其转换为简化的内部格式：

```typescript
// 原始 B站弹幕
{
  "cmd": "DANMU_MSG",
  "info": [
    0,              // 弹幕类型
    1,              // 弹幕模式
    16777215,      // 颜色
    "用户名",       // 用户名
    0,              // 等级
    "",             // 弹幕内容
    123456,         // 时间戳
    123456789,      // 用户UID
    ...             // 其他字段
  ]
}

// 转换为内部格式
{
  type: "danmaku",
  data: {
    id: "bilibili_123456789_1700000000",
    uid: 123456789,
    user: "用户名",
    content: "弹幕内容",
    timestamp: 1700000000,
    color: "#FFFFFF"
  }
}
```

### 步骤 3: 管理员指令处理

```
┌─────────────────────────────────────────────────────────────────┐
│                  管理员指令处理流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  弹幕内容: "/sleep"                                            │
│         │                                                       │
│         ▼                                                       │
│  ┌─────────────────────┐                                       │
│  │ is_admin(uid)?      │ ──── 是 ──▶ 继续检查                  │
│  └──────────┬──────────┘                                       │
│             │ 否                                                │
│             ▼                                                  │
│  跳过管理员指令处理                                             │
│         │                                                       │
│         ▼                                                       │
│  ┌─────────────────────┐                                       │
│  │ is_admin_command()? │ ──── 是 ──▶ 执行指令                  │
│  └──────────┬──────────┘                                       │
│             │ 否                                                │
│             ▼                                                  │
│  返回 false，继续后续流程                                        │
│                                                                 │
│  管理员指令列表:                                                │
│  - /sleep   : AI 进入休眠                                      │
│  - /wake    : AI 唤醒                                          │
│  - /pause   : 暂停音乐                                          │
│  - /resume  : 恢复音乐                                          │
│  - /next    : 切歌                                              │
│  - /rm      : 移除当前歌曲                                      │
│  - /volume  : 设置音量                                          │
│  - /hide    : 隐藏管理员弹幕                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 步骤 4: 点歌请求拦截

```
┌─────────────────────────────────────────────────────────────────┐
│                  点歌请求拦截流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  弹幕内容: "点歌 歌曲名" 或 "点歌 BV1xx..."                    │
│         │                                                       │
│         ▼                                                       │
│  ┌─────────────────────┐                                       │
│  │ is_music_request()? │ ──── 是 ──▶ 继续处理                  │
│  └──────────┬──────────┘                                       │
│             │ 否                                                │
│             ▼                                                  │
│  返回 false，后续交给 AI 处理                                    │
│         │                                                       │
│         ▼                                                       │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 1. 检查用户点歌数量限制 (最多2首)                         │ │
│  │ 2. 检查队列长度限制 (最多5首)                             │ │
│  │ 3. 提取 BV 号或歌名                                        │ │
│  │ 4. 搜索歌曲:                                              │ │
│  │    - 预备歌单匹配                                         │ │
│  │    - UP 主视频匹配                                        │ │
│  │    - LLM 验证 (非预备歌单时)                              │ │
│  │ 5. 获取音频 URL                                           │ │
│  │ 6. 加入播放队列                                           │ │
│  └──────────────────────────────────────────────────────────┘ │
│         │                                                       │
│         ▼                                                       │
│  广播点歌结果:                                                  │
│  - 成功: {type: "music_added", data: {...}}                   │
│  - 失败: {type: "music_error", data: {error: "..."}}           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 步骤 5: 广播弹幕到前端

无论弹幕是否被拦截处理，最终都会通过 WebSocket 广播到前端：

```json
{
  "type": "danmaku",
  "data": {
    "id": "bilibili_123456_1700000000",
    "uid": 123456,
    "user": "用户名",
    "content": "弹幕内容",
    "timestamp": 1700000000
  }
}
```

### 步骤 6: AI 对话处理

```
┌─────────────────────────────────────────────────────────────────┐
│                  AI 对话处理流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  普通弹幕（非管理员指令、非点歌请求）                            │
│         │                                                       │
│         ▼                                                       │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 1. brain.push_danmaku()                                  │ │
│  │    - 弹幕加入缓冲区                                       │ │
│  │    - 过滤检查 (sleep 模式)                               │ │
│  │    - 检查回复间隔                                         │ │
│  └──────────────────────────────────────────────────────────┘ │
│         │                                                       │
│         │ accepted                                             │
│         ▼                                                       │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 2. brain.try_reply()                                     │ │
│  │    - 获取用户记忆上下文                                  │ │
│  │    - 构建 prompt                                          │ │
│  │    - 调用 LLM 生成回复                                   │ │
│  │    - 调用 TTS 合成语音                                   │ │
│  └──────────────────────────────────────────────────────────┘ │
│         │                                                       │
│         │ 广播 AI 回复                                         │
│         ▼                                                       │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 3. 广播回复到前端                                        │ │
│  │    - start: 开始回复                                     │ │
│  │    - text: 文字内容                                      │ │
│  │    - audio: 音频数据                                     │ │
│  │    - end: 结束回复                                       │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
