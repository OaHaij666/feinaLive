# 技术栈说明

## 技术选型总览

飞娜直播间采用现代化的技术栈，以前后端分离架构为基础，结合AI技术实现智能直播功能。

## 后端技术栈

### Web框架

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| **FastAPI** | 0.1.0 | 高性能异步框架，原生支持 WebSocket，自动生成 API 文档 |
| **Uvicorn** | - | ASGI 服务器，支持热重载 |
| **Pydantic** | - | 数据验证，与 FastAPI 深度集成 |

**FastAPI 选型理由**：
1. **异步原生**：基于 Starlette，天然支持 async/await
2. **类型提示**：自动生成 OpenAPI 文档
3. **WebSocket 支持**：无需额外配置即可使用 WebSocket
4. **性能优异**：接近 Go/Node.js 的性能水平

### 数据库

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| **MySQL** | 8.0+ | 成熟稳定，支持高并发，生态完善 |
| **SQLAlchemy** | 2.x | 强大的 ORM，支持异步操作 |
| **aiomysql** | - | MySQL 的异步驱动 |

**MySQL 选型理由**：
1. **成熟稳定**：经过大量生产环境验证
2. **高并发支持**：连接池+异步驱动支持高并发场景
3. **生态完善**：丰富的运维工具和文档
4. **功能丰富**：支持事务、存储过程等高级特性

### AI 相关

| 技术 | 版本 | 用途 |
|-----|------|------|
| **LangGraph** | - | AI 对话流程编排，状态机管理 |
| **ONNX Runtime** | - | 模型推理加速 |
| **bilibili-api** | - | B站 API 封装 |

**LangGraph 选型理由**：
1. **流程可视化**：对话流程清晰可控
2. **状态管理**：内置状态机，便于复杂流程
3. **可扩展**：支持自定义节点和边

### 异步编程

```python
# 异步架构示例
async def process_danmaku():
    # 并发处理多个任务
    await asyncio.gather(
        handle_music_request(),
        handle_ai_reply(),
        broadcast_to_frontend()
    )
```

**异步设计原则**：
1. 所有 I/O 操作使用 async/await
2. 使用 asyncio.Lock 保护共享资源
3. 避免阻塞操作

## 前端技术栈

### 框架

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| **Vue 3** | 3.x | 组合式 API，更好的 TypeScript 支持 |
| **TypeScript** | 5.x | 类型安全，开发体验好 |
| **Vite** | 5.x | 极速开发体验，HMR 快 |

**Vue 3 选型理由**：
1. **组合式 API**：逻辑复用更方便
2. **响应式优化**：性能更好
3. **生态成熟**：组件库丰富

### 状态管理

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| **Pinia** | - | Vue 3 官方推荐，类型支持好 |
| **Composables** | - | 逻辑复用，替代 Mixins |

**Pinia 状态管理示例**：
```
stores/
├── danmaku.ts    # 弹幕状态
├── music.ts      # 音乐状态
├── llm.ts        # AI 对话状态
└── stream.ts     # 流状态
```

### 样式

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| **Tailwind CSS** | 3.x | 原子化 CSS，开发效率高 |
| **PostCSS** | - | CSS 后处理 |

## 基础设施

### 流媒体

| 组件 | 用途 |
|-----|------|
| **nginx-rtmp** | RTMP 推流服务器 |
| **HLS** | HTTP 直播流分发 |

**nginx-rtmp 配置要点**：
```
rtmp {
    server {
        listen 1935;
        application live {
            live on;
            record off;
        }
    }
}
```

### GPU 加速

| 技术 | 用途 |
|-----|------|
| **CUDA** | NVIDIA GPU 加速 |
| **ONNX Runtime** | 模型推理优化 |
| **TensorRT** | 可选的进一步加速 |

**GPU 加速流程**：
```
模型文件 (.onnx)
      │
      ▼
ONNX Runtime (CUDA Execution Provider)
      │
      ▼
GPU 推理
      │
      ▼
渲染输出
```

## 外部服务集成

### B站 API

| 接口 | 用途 |
|-----|------|
| 弹幕服务器 | WebSocket 实时弹幕 |
| 视频信息 API | 获取视频元数据 |
| 音频流 API | 获取音频直链 |

**认证方式**：SESSDATA Cookie

### LLM 服务

| 服务商 | 模型 | 用途 |
|-------|------|------|
| 豆包 | doubao-seed-character | AI 主播对话 |
| DeepSeek | deepseek-chat | 备选模型 |
| OpenAI 兼容 | - | API 格式兼容 |

**LLM 调用配置**：
```yaml
llm:
  api_url: "https://api.example.com/v1"
  api_key: "your-api-key"
  model: "doubao-seed-character"
  temperature: 0.7
  max_tokens: 200
```

### TTS 服务

| 服务商 | 功能 |
|-------|------|
| 火山引擎 | 声音复刻，高质量语音合成 |

**TTS 配置**：
```yaml
volcano:
  appid: "your-appid"
  access_token: "your-token"
  speaker_id: "your-voice-id"
```

## 技术债务与未来规划

### 当前技术债务

1. **数据库**：SQLite 在高并发场景下可能成为瓶颈
2. **缓存**：缺少 Redis 缓存层
3. **监控**：缺少 APM 监控

### 未来规划

| 方向 | 计划 |
|-----|------|
| 数据库 | 迁移到 PostgreSQL |
| 缓存 | 引入 Redis |
| 容器化 | Docker + K8s 部署 |
| 监控 | Prometheus + Grafana |
