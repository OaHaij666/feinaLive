# 系统整体架构

## 架构概览

飞娜直播间采用**前后端分离**的微服务架构，后端基于 FastAPI 构建，前端使用 Vue 3。系统通过 WebSocket 实现实时双向通信，支持弹幕互动、AI对话、音乐播放和虚拟形象渲染。

## 核心组件说明

### 1. 接入层

#### FastAPI Application
- **端口**：8000
- **职责**：HTTP API 和 WebSocket 接入
- **路由模块**：
  - `/bilibili/*` - B站弹幕相关
  - `/music/*` - 音乐播放相关
  - `/ai/*` - AI对话相关
  - `/avatar/*` - 虚拟形象控制

#### WebSocket Manager
- **类**：`ConnectionManager`
- **职责**：管理 WebSocket 连接，按房间分发消息
- **特点**：支持多房间、线程安全

### 2. 业务逻辑层

#### danmaku_handler.py（核心协调器）
这是整个系统的消息分发中心，负责：
1. 接收弹幕消息
2. 判断是否为管理员指令
3. 判断是否为点歌请求
4. 普通弹幕推送到AI大脑

```
弹幕消息 ──┬── 管理员指令 ──▶ AdminCommandHandler
           │
           ├── 点歌请求 ──▶ DanmakuMusicService ──▶ MusicQueue
           │
           └── 普通弹幕 ──▶ AIHostBrain ──▶ LLM + TTS
```

#### AI对话模块
- **核心类**：`AIHostBrain`
- **流程编排**：LangGraph 状态机
- **记忆架构**：
  - 短期：SessionHistory（50条）
  - 中期：UserProfile（用户画像）
  - 长期：RAG（知识库）

### 3. 渲染引擎层

#### EasyVtuber Manager
- **输入源**：鼠标位置、音频电平、人脸检测
- **模型**：THA3/THA4（Talking Head Animation）
- **推理**：ONNX Runtime + CUDA
- **后处理**：RIFE 插帧、超分辨率
- **输出**：WebSocket PNG 流

### 4. 数据存储层

| 存储类型 | 用途 | 关键数据 |
|---------|------|---------|
| SQLite | 业务数据 | 用户画像、播放列表、UP主视频 |
| config.yaml | 配置数据 | API密钥、模型参数 |
| 模型文件 | AI模型 | THA3/THA4、RIFE、SR |

## 关键设计决策

### 1. 异步架构
- 全异步设计（asyncio）
- 非阻塞 I/O 操作
- 并发处理多个弹幕消息

### 2. 单例模式
- `get_music_queue()` - 音乐队列
- `get_host_brain()` - AI大脑
- `get_tts_client()` - TTS客户端
- `get_easyvtuber_manager()` - 虚拟形象管理器

### 3. 事件驱动
- 弹幕事件触发处理流程
- 管理员指令触发状态变更
- 音乐播放完成触发下一首

### 4. 流式处理
- LLM 流式输出
- TTS 按句子合成
- 虚拟形象实时渲染
