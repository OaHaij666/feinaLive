# 模块间交互关系

## 模块通信概览

飞娜直播间的模块间通信主要通过以下方式实现：

1. **直接函数调用**：模块内部通信
2. **WebSocket**：前后端实时通信
3. **HTTP API**：RESTful 接口
4. **共享状态**：单例模式共享数据

## 核心交互流程

### 1. 弹幕处理流程

```
用户发送弹幕
      │
      ▼
┌─────────────────┐
│ B站弹幕服务器   │
└────────┬────────┘
         │ WebSocket
         ▼
┌─────────────────┐
│ BilibiliClient  │  ──────── 解析弹幕消息
│ (bilibili模块)  │
└────────┬────────┘
         │ on_message 回调
         ▼
┌─────────────────┐
│  router.py      │  ──────── WebSocket 路由
│  /ws/{room_id}  │
└────────┬────────┘
         │ 调用
         ▼
┌─────────────────────────────────────────────────────────────┐
│                    process_danmaku()                        │
│                    (danmaku_handler.py)                     │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │ 1. 检查是否为管理员指令                              │  │
│   │    └─▶ AdminCommandHandler.handle()                 │  │
│   │                                                      │  │
│   │ 2. 检查是否为点歌请求                                │  │
│   │    └─▶ DanmakuMusicService.process_danmaku()        │  │
│   │        └─▶ MusicQueue.add()                         │  │
│   │                                                      │  │
│   │ 3. 普通弹幕处理                                      │  │
│   │    └─▶ AIHostBrain.push_danmaku()                   │  │
│   │        └─▶ AIHostBrain.try_reply()                  │  │

              ┌────────────────────────┐

              └────────────────────────┘
```

## 数据流向总结

| 数据类型 | 来源 | 处理模块 | 目标 |
|---------|------|---------|------|
| 弹幕消息 | B站服务器 | bilibili → danmaku_handler | 前端/AI |
| 点歌请求 | 弹幕 | danmaku_handler → music | 播放队列 |
| AI回复 | AI模块 | ai → TTS | 前端音频 |
| 虚拟形象 | 渲染引擎 | easyvtuber | 前端图片 |
| 管理指令 | 弹幕 | danmaku_handler → admin | 状态变更 |
